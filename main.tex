    Usando el IDE Thonny; escribir el siguiente código y comentar cada instrucción:

    \subsection*{Propuesta de solución}
    Para lograr el parpadeo básico de un LED, se propone un script en MicroPython que manipula directamente los registros de hardware del microcontrolador. La solución requiere importar el módulo \texttt{machine} para tener acceso a la configuración de los pines físicos, y el módulo \texttt{utime} para generar bloqueos de tiempo en el procesador. Se configura el pin GPIO 0 como una salida digital. Posteriormente, mediante un bucle infinito \texttt{while True}, se alterna el estado lógico del pin enviando un nivel alto (1) y un nivel bajo (0), intercalando retardos de 500 milisegundos entre cada cambio de estado para hacer el parpadeo perceptible al ojo humano.

    El programa inicia importando las bibliotecas necesarias para el control de hardware (\texttt{machine}) y la gestión de temporización (\texttt{utime}). A continuación, se configura el pin GPIO~0 como una salida digital, lo que permite al microcontrolador enviar señales eléctricas a través de este terminal. El flujo entra entonces en un ciclo infinito donde se ejecutan cuatro operaciones secuenciales de forma ininterrumpida: primero se establece el nivel lógico alto (3.3\,V) en el pin, encendiendo el LED; luego el procesador se bloquea durante 500\,ms; después se establece el nivel lógico bajo (0\,V), apagando el LED; y finalmente se ejecuta otro bloqueo de 500\,ms. Al concluir estas cuatro instrucciones, el flujo retorna al inicio del ciclo, generando una señal cuadrada periódica de 1\,Hz que se refleja como un parpadeo constante del LED.

    La ejecución del script confirma el control directo sobre el hardware del microcontrolador. La instrucción \texttt{machine.Pin(0, machine.Pin.OUT)} configura correctamente el pin físico asociado al GPIO 0, estableciendo sus registros internos para operar como salida digital. 

    Al entrar en el ciclo \texttt{while True}, la instrucción \texttt{LED.value(1)} satura la salida, colocando un voltaje de 3.3V (nivel lógico alto) en el pin GPIO 0, lo que enciende el LED. Inmediatamente después, \texttt{utime.sleep\_ms(500)} detiene la ejecución del procesador durante exactamente 500 milisegundos. Transcurrido este tiempo, \texttt{LED.value(0)} drena el voltaje a 0V (nivel lógico bajo), apagando el componente, seguido de otra pausa de 500 milisegundos. Este comportamiento secuencial y cíclico genera una onda cuadrada perfecta con un periodo total de 1000 milisegundos (1 segundo), logrando una frecuencia de parpadeo estable de 1 Hz.

    \section*{Actividad 2}

    Realizar las modificaciones necesarias para que los tiempos sean más pequeños y más grandes que el código anterior.

     \subsection*{Propuesta de solución}
    Para alterar la frecuencia de parpadeo dentro de un mismo ciclo de ejecución, se mantiene la configuración inicial del pin GPIO 0 como salida digital, pero se expande la lógica dentro del bucle principal. La propuesta divide el ciclo en dos fases: una de alta frecuencia y otra de baja frecuencia. Para los tiempos más pequeños, se establecen los cambios de estado del pin con retardos de 100 milisegundos. Para los tiempos más grandes, se añaden instrucciones idénticas de cambio de estado, pero acompañadas de retardos extendidos de 2000 milisegundos (2 segundos). Ambas fases se ejecutan secuencialmente dentro del bucle infinito.

    TEl programa ingresa a un bucle infinito dividido en dos fases claramente diferenciadas. En la \textit{fase rápida}, el pin alterna entre nivel alto y nivel bajo con retardos de apenas 100\,ms, produciendo un parpadeo veloz con un periodo de 200\,ms (5\,Hz). Inmediatamente después, sin interrumpir el flujo, se ejecuta la \textit{fase lenta}, donde los mismos cambios de estado se acompañan de retardos de 2000\,ms, generando un periodo de 4000\,ms (0.25\,Hz). Dado que ambas fases están concatenadas dentro del mismo \texttt{while True}, el flujo de datos es estrictamente secuencial: el microcontrolador completa la fase rápida antes de iniciar la lenta, y al finalizar la fase lenta, retorna al inicio del ciclo repitiendo la secuencia indefinidamente.

    \textbf{Análisis de resultados}

    La modificación de los parámetros en la función \texttt{utime.sleep\_ms()} evidencia la naturaleza bloqueante de esta instrucción sobre el hilo de ejecución principal del microcontrolador. Durante la primera fase del bucle, los retardos de 100 milisegundos fuerzan al pin GPIO 0 a conmutar rápidamente entre 3.3V y 0V. Esto genera un tren de pulsos muy corto con un periodo de 200 milisegundos, visualizándose físicamente como un parpadeo o estrobo rápido.

    Posteriormente, el flujo del programa avanza a la segunda fase, donde las instrucciones \texttt{utime.sleep\_ms(2000)} obligan al procesador a mantener el estado lógico actual del pin durante 2 segundos completos antes de ejecutar la siguiente línea de código. Esto crea un periodo de 4000 milisegundos (4 segundos), resultando en un encendido y apagado prolongado. La combinación de ambas rutinas en el mismo \texttt{while True} demuestra que los tiempos de retardo dictan absolutamente la anchura de los pulsos generados, permitiendo crear secuencias asimétricas de temporización sobre el mismo componente de hardware sin necesidad de usar temporizadores (\textit{timers}) por interrupción.
