    \subsection*{Propuesta de solución}
    El objetivo es monitorear el estado físico de un interruptor de palanca e imprimir su estado en la consola. La solución emplea la clase \texttt{Pin} del módulo \texttt{machine} para configurar el terminal GPIO 8 del microcontrolador estrictamente como una entrada digital (\texttt{Pin.IN}). Mediante un bucle infinito \texttt{while True}, se realiza un sondeo continuo (\textit{polling}) del estado del pin. Se implementa una estructura condicional \texttt{if-else} que evalúa el nivel lógico retornado por el método \texttt{value()}; si se detecta un nivel lógico alto (1), se notifica por consola que el circuito está cerrado; de lo contrario (0), se notifica como abierto. Se intercala un retardo de 500 milisegundos para evitar la saturación del procesador y estabilizar las lecturas en la consola serial.

    El flujo de datos inicia con la importación de los módulos \texttt{machine} y \texttt{time}, seguido de la configuración del pin GPIO~8 como entrada digital. El programa entra entonces en un bucle infinito donde, en cada iteración, lee el nivel lógico presente en el pin. Si el valor leído es 1 (3.3\,V), el flujo toma la rama afirmativa e imprime en consola que el interruptor está cerrado; si el valor es 0 (0\,V), toma la rama negativa e imprime que está abierto. Ambas ramas convergen en un retardo de 500\,ms antes de regresar a la lectura, estableciendo así un ciclo de sondeo continuo a 2\,Hz.

    \textbf{Análisis de resultados}

    La ejecución de este código demuestra la capacidad del microcontrolador para leer señales eléctricas externas y traducirlas a lógica de software. La instrucción \texttt{sw1\_1 = Pin(8, Pin.IN)} configura los registros internos del procesador para deshabilitar los transistores de salida en ese terminal y habilitar el búfer de lectura, permitiendo sensar el voltaje externo. 

    Durante el ciclo infinito, cuando el interruptor físico cierra el circuito, el pin 8 recibe un voltaje de 3.3V. El hardware interpreta este voltaje como un nivel lógico alto, causando que \texttt{sw1\_1.value()} sea igual a 1, lo cual activa la primera rama de la condicional y envía la cadena de texto a la consola. Inversamente, al abrir el interruptor, el voltaje decae a 0V, la lectura retorna 0 y el control de flujo salta a la rama \texttt{else}. La inclusión de \texttt{time.sleep(0.5)} establece una frecuencia de muestreo de 2 Hz, lo que resulta suficiente para capturar las interacciones mecánicas humanas de manera estable sin desbordar el búfer de salida serial del IDE.

    \subsection*{Propuesta de solución}

    Para integrar una respuesta física ante el estímulo del interruptor, se expande el código anterior añadiendo una etapa de salida digital. Se inicializa el terminal GPIO 0 como salida (\texttt{Pin.OUT}), el cual está eléctricamente conectado a un LED verde en la tarjeta de desarrollo. La lógica del programa se modifica vinculando el estado de este LED al estado de lectura del interruptor: dentro de la condición donde se detecta el nivel alto (interruptor cerrado), se envía un 1 lógico al LED para encenderlo; en la condición de nivel bajo (interruptor abierto), se envía un 0 lógico para apagarlo. El sondeo se mantiene con un retardo de medio segundo.

    El programa evidencia la interconexión de periféricos de entrada y salida (I/O) mediados por la Unidad Lógico Aritmética del microcontrolador. Al instanciar \texttt{led\_verde = Pin(0, Pin.OUT)}, se configura un canal de salida capaz de suministrar voltaje. 

    El bloque condicional funciona ahora como un puente directo entre ambos pines físicos. Cuando la condición \texttt{sw1\_1.value() == 1} se cumple, la instrucción \texttt{led\_verde.value(1)} ordena al hardware colocar 3.3V en el terminal GPIO 0, cerrando el circuito del LED a tierra y provocando su iluminación simultánea con la impresión en consola. Por el contrario, cuando la lectura es 0, \texttt{led\_verde.value(0)} anula el voltaje de salida, apagando el componente. Esta sincronía demuestra que la respuesta del actuador (el LED) está estrictamente gobernada por el estado lógico del sensor (el interruptor), logrando un control de lazo abierto simple y altamente predecible.
